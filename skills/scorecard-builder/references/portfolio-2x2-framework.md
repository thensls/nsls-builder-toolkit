# The Project-Portfolio 2×2 Framework

*The lens behind Side A, especially for operating / coordination / leadership roles. Use it to sort candidate work into accountabilities and to align a report on how their priorities get chosen.*

For many roles — anything where the person coordinates a shifting field of possible work rather than executing one fixed thing — the strongest top accountability is **"run the right portfolio."** This framework gives that accountability a shared language.

## The two axes → exactly four categories

Every project sits on two axes:

- **Advance vs. Protect** — are you moving the state forward, or defending it?
- **Output vs. Machine** — is it about the value produced, or the operating system that produces it?

|  | **Output** — the value we produce | **Machine** — the system that produces it |
|---|---|---|
| **Advance** | ① **Growth driver** — move a growth number up | ② **Operating efficiency** — make the machine leaner |
| **Protect** | ③ **Hygiene** — external / sudden harm (legal, security, FERPA) | ④ **Reliability** — internal decay of what already works |

## The four categories

| Category | The bet | What qualifies | Prioritization question |
|---|---|---|---|
| **① Growth driver** | *Upside* — move a growth number | Directly moves members, revenue, activation, conversion, engagement | Which few drivers, if landed this quarter, move the number the most? |
| **② Operating efficiency** | *Leverage* — do more with the same | Makes the machine run better: systems, process, coordination, less rework | Which fix unlocks or de-risks the most *other* work? |
| **③ Hygiene** | *Floor (external)* — don't get hurt | Protects from sudden / existential downside: legal, security, compliance, privacy | Which exposures, if left open, could halt or damage the business? |
| **④ Reliability** | *Floor (internal)* — don't let it rot | Deliberate upkeep that keeps existing systems & commitments from decaying — migrations, renewals, reliability pushes *(not routine daily ops)* | What is quietly decaying that will break if we keep deferring it? |

## The three insights that make it a judgment tool, not a filing system

1. **Rows are offense vs. defense.** ①② create new value; ③④ protect existing value. Hygiene and reliability are *both* floors — they feel like the same thing because they're the two "protect" cells — but they defend against different failures: **hygiene = sudden external harm** (a breach, a lawsuit, a FERPA violation); **reliability = gradual internal decay** (systems drift, renewals lapse, bugs compound). Naming reliability on its own is how it gets *funded on purpose* instead of silently losing every contest to growth until something breaks (the Kerr "hard-to-see work goes unrewarded" failure mode).

2. **Floors are cleared, not ranked.** You don't skip a FERPA fix or let a core system rot because a growth project "scored higher." Naive ROI-ranking across all four is the mistake the frame exists to prevent. Clear the floors; then optimize upside (① growth) and leverage (② efficiency).

3. **Bottlenecks cut across all four.** A "bottleneck" is any project — in any quadrant — that *halts other priorities if it isn't done*. It jumps the queue regardless of category. (Example: an auth integration can be both a growth driver *and* a bottleneck, because downstream product metrics stay blocked until it lands.)

Within each category, most candidates are deferrable. The core skill of the role is naming the few that actually move the category's number — and protecting them from the noise of everything that merely *could* be done.

## How to use it in a scorecard

- **As Accountability #1** for a portfolio/coordination role: "Runs the operating portfolio — sorts work into the four categories, identifies the drivers and bottlenecks, mitigates cross-team risk, and shepherds the vital few to done." The Pass/Fail measure is a live ranked portfolio + named driver/bottleneck projects landing on committed dates.
- **As a classification tag** on every accountability, so the card (and later a portfolio tracker) can be read by category and KPIs tracked along the four lines.
- **As the alignment conversation itself** — walking a report through the 2×2 tells them *why* the categories exist, not just what they are. That's what gets manager and report on the same page fast.

*This is a shared working framework, not a graded formula — expect to revise it with the report as it gets used. Origin: developed 2026-07-11 building the Head-of-Operations scorecard; Kevin's durable model for prioritizing and tracking projects/KPIs org-wide.*
